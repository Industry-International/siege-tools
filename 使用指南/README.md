## 1. 方块总览

| 方块 | 功能 |
|------|------|
| `siege_tools:vehicle_deployer` | 载具部署台 — 自动/手动召唤 SBW/MCSP 载具 |
| `siege_tools:ammo_station` | 弹药补给站 — 自动扫描并补充载具弹药 |

两个方块均位于创造模式物品栏的 **功能方块** 标签页。

## 2. 载具部署台

### 2.1 放置与初始化

部署台放置后自动初始化配置（默认 30 秒自动重生、生成带弹药）。

### 2.2 GUI 配置

**OP (权限等级 ≥ 2) 右键** 打开配置 GUI：

```
┌──────────────────────┐
│  §6╔ 载具部署台 ╗   │
├──────────────────────┤
│ [载具][基础][坐标]    │  ← TabView 页签
│ [⚙NBT][⚡NBT上]     │
├──────────────────────┤
│  当前页签配置内容     │
├──────────────────────┤
│ [✔ 保存][↻ 重置]     │
│ [⚡ 立即部署]         │
└──────────────────────┘
```

#### 页签说明

| 页签 | 字段 | 说明 |
|------|------|------|
| **载具** | 类别/载具/ID | 两级联动下拉选择，或直接输入完整实体 ID |
| **基础** | 重生延迟 | tick 数，20 tick = 1 秒 |
| | 自动重生 | 1=开启，载具摧毁后自动召唤 |
| | 带弹药 | 1=生成时携带弹药 |
| **坐标** | X/Y/Z 偏移 | 部署时相对于方块的位置偏移 |
| | 朝向/俯仰 | 载具生成时的方向 |
| **⚙NBT** | Energy/Health | 核心属性快速修改 |
| | Invulnerable | 无敌模式 |
| | DecoyReady | 诱饵弹状态 |
| | ⟳ 应用默认值 | 从数据库模板填充 NBT 字段 |
| **⚡NBT上** | deployNBT JSON | 完整的自定义 NBT 模板 |

### 2.3 生命周期

```
放置方块 → 初始化
    ↓
配置载具类型 + 参数
    ↓
[自动重生=1] → 冷却结束后自动召唤
[自动重生=0] → 点击「立即部署」手动召唤
    ↓
载具存活 → 追踪 UUID
    ↓
载具被摧毁
    ↓
死亡事件反查 → 清 UUID → 开始冷却
    ↓
冷却结束 → 自动部署（如开启）
```

### 2.4 冷却重启保护

服务器重启后 `cooldownEnd` 超过当前时间 1 小时会**自动重置为 0**，防止残留旧数据导致的无限冷却。

## 3. 弹药补给站

### 3.1 放置与初始化

放置后自动从弹药注册表获取所有弹药类型，默认每种 64 个。

### 3.2 GUI 配置

**OP 右键** 打开配置 GUI：

```
┌──────────────────────────────┐
│    §6╔ 弹药补给站 ╗         │
├──────────────────────────────┤
│ ◀ [基础][炮弹][小口][导弹] ▶ │  ← 页签切换
├──────────────────────────────┤
│    当前分类弹药输入框         │
├──────────────────────────────┤
│  [✔ 保存配置] [↻ 重置默认]   │
└──────────────────────────────┘
```

#### 页签说明

| 页签 | 内容 |
|------|------|
| **基础** | 扫描范围、冷却时间、驶入等待时间 |
| **炮弹/小口径/...** | 弹药分类（从 `_ammo_types.json` 自动读取） |
| **作弊** | 切换作弊模式（需权限） |

#### 弹药编辑

每个弹药类型一行，格式：`显示名: [输入框]`，2 列网格布局。

### 3.3 补给流程

```
补给站每 20 tick 扫描一次 AABB 范围
    ↓
发现 SBW/MCSP 载具 → 记录进入时间
    ↓
载具停留 ≥ enterDelay 秒
    ↓
查数据库获取车辆所需的弹药类型（defaultAmmo）
    ↓
按站配上限补充（不超 64 一组）
    ↓
/ 命令写回实体 NBT
    ↓
设置冷却
```

**补给原则：**
- 只补给载具 JSON 中 `defaultAmmo` 定义的弹药类型
- 如车辆无 `defaultAmmo` → 使用全部站配弹药兜底
- 每次补给不超过配置的最大值

### 3.4 进度动画

载具停留在范围内时，驾驶员动作栏显示：

```
§e弹药补给中 §a45% §a■■■■■■■■■§8□□□□□□□□□□□
```

圆环符号表示进度阶段：`○◔◑◕●`

## 4. 数据文件配置

数据文件位于：`src/main/resources/data/siege_tools/sbw_vehicle_db/`

### 4.1 目录结构

```
sbw_vehicle_db/
├── _registry.json          ← 分类注册表
├── _ammo_types.json        ← 弹药类型注册表
├── main_battle_tank/       ← 主战坦克分类目录
│   ├── superbwarfare--t_90.json
│   └── ...
├── aircraft/               ← 飞机分类目录
├── helicopter/             ← 直升机分类目录
└── ...                     ← 其他分类
```

### 4.2 注册表

#### `_registry.json`

```json
{
  "formatVersion": 1,
  "_comment": "分类注册表",
  "categories": {
    "main_battle_tank": {
      "displayName": "主战坦克",
      "icon": "superbwarfare:t_90"
    }
  }
}
```

#### `_ammo_types.json`

```json
{
  "formatVersion": 1,
  "ammoTypes": {
    "large_shell_ap": {
      "id": "superbwarfare:large_shell_ap",
      "displayName": "§6大口径AP弹",
      "enName": "Large AP Shell",
      "maxStack": 64,
      "category": "炮弹"
    }
  }
}
```

字段说明：

| 字段 | 必填 | 说明 |
|------|------|------|
| `id` | ✅ | 物品完整注册名 |
| `displayName` | ❌ | 显示名称（支持 § 颜色码） |
| `enName` | ❌ | 英文显示名 |
| `maxStack` | ❌ | 最大堆叠（默认 64） |
| `category` | ❌ | 弹药分类名，用于 GUI 分组 |

### 4.3 载具 JSON

每辆载具一个 JSON 文件，文件名格式：`模组名--车辆id.json`

```json
{
  "formatVersion": 1,
  "vehicleId": "superbwarfare:t_90",
  "mod": "superbwarfare",
  "displayType": "坦克",
  "hudType": "tank",
  "stats": {
    "maxHealth": 1200,
    "maxEnergy": 10000,
    "mass": 46.0
  },
  "parts": ["turret", "hull", "track_left", "track_right"],
  "weapons": [
    {
      "key": "cannon",
      "displayKey": "主炮",
      "ammoTypes": ["large_shell_ap", "large_shell_he"],
      "magazine": 1,
      "rpm": 6,
      "damage": 250.0
    }
  ],
  "defaultAmmo": {
    "superbwarfare:large_shell_ap": 32,
    "superbwarfare:large_shell_he": 32
  },
  "spawnNbt": {
    "Energy": 10000,
    "Health": 1200
  }
}
```

> **注意：** `defaultAmmo` 的 key 使用 **完整物品 ID**（带命名空间），补给站会自动转换为短名匹配配置。

| 字段 | 必填 | 说明 |
|------|------|------|
| `formatVersion` | ✅ | 数据格式版本（当前为 1） |
| `vehicleId` | ✅ | 实体注册名，如 `superbwarfare:t_90` |
| `mod` | ❌ | 来源模组（如 `superbwarfare`, `mcsp`） |
| `displayType` | ❌ | GUI 显示类型 |
| `hudType` | ❌ | HUD 类型标识 |
| `stats` | ❌ | 载具属性（maxHealth, maxEnergy, mass） |
| `parts` | ❌ | 载具部件列表 |
| `weapons` | ❌ | 武器配置列表 |
| `defaultAmmo` | ❌ | 弹药最大储量（补给站据此补充） |
| `spawnNbt` | ❌ | 生成 NBT 模板 |

### 4.4 自动发现机制

数据文件通过**目录结构自动发现**：
- 每个子目录名 = 分类 key
- 目录下的 `.json` 文件 = 该分类的载具
- 文件名中 `--` 分隔模组名和车辆 ID

## 5. 常见问题

### Q: 部署台不自动重生？
A: 检查 GUI 中「自动重生」设置为 1，且冷却未结束。服务器重启后冷却异常会自动重置。

### Q: 弹药补给站不补充弹药？
A: 检查：
1. 载具是否在扫描范围内
2. 载具是否停留超过驶入等待时间
3. 站配 slots 是否有该弹药的配置值（>0）
4. 载具 JSON 的 `defaultAmmo` 是否包含该弹药类型

### Q: 如何添加新弹药类型？
A: 在 `_ammo_types.json` 的 `ammoTypes` 中添加条目，重启服务器。GUI 会自动识别并显示。

### Q: 如何添加新载具？
A: 在对应分类目录下创建 JSON 文件，重启服务器。部署台下拉列表会自动更新。

### Q: 普通玩家右键方块看到什么？
A: 非 OP 玩家右键部署台会看到状态信息（类型、状态、冷却）。补给站暂未开放普通玩家查看。
