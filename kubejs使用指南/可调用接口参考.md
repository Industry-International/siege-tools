## 全局调用规则

在 KubeJS 7 (Rhino) 中：

| 声明方式 | 作用域 | 是否可被其他文件访问 |
|---------|--------|-------------------|
| `const` (顶层) | 全局 | ✅ 其他所有文件可直接读写 |
| `var` (顶层) | 全局 | ✅ 可重复声明，但应避免 |
| `function` | 全局 | ✅ 等同 var |
| `global.*` | 跨 startup/server | ✅ 全局共享，不能传函数引用 |
| `let` / `var` (函数内) | 局部 | ❌ 仅在函数内可见 |

> ⚠ **警告**：已用 `const` 声明的变量不可在其他文件中用 `const`/`var`/`let` 重复声明，否则报 `redeclaration error`。
> 详细已声明列表见 `给AI阅读的文档/const.txt`。

## 模块管理器

**代码路径：** `server_scripts/module_manager/`

### 可调用的常量

| 常量名 | 类型 | 说明 |
|-------|------|------|
| `MODULE_REGISTRY` | `Array<{id, name, default}>` | 已注册的模块列表 |
| `MODULE_FLAG_PREFIX` | `string` | 模块持久化键前缀 (`module_`) |

### 可调用的函数

| 函数签名 | 说明 | 调用示例 |
|---------|------|---------|
| `isModuleEnabled(server, moduleId)` | 判断指定模块是否已启用 | `isModuleEnabled(event.server, 'team_revive')` |
| `setModuleEnabled(server, moduleId, enabled)` | 设置模块启用状态 | `setModuleEnabled(server, 'my_mod', true)` |
| `findModule(id)` | 按 ID 查找模块注册记录 | `findModule('team')` |

### 注册新模块

在 `a_registry.js` 的 `MODULE_REGISTRY` 中添加：

```js
{ id: 'your_module', name: '显示名称', default: false }
```

然后在对应脚本中用 `isModuleEnabled(server, 'your_module')` 判断启用状态。

## 职业系统

**代码路径：** `server_scripts/profession/`

### 可调用的常量

| 常量名 | 类型 | 说明 |
|-------|------|------|
| `PROF_CONFIGS` | `Object` | 所有职业的完整武器配置（基础数据源） |
| `PROF_TAG_LIST` | `Array<string>` | 职业标签列表 `['assault','scout','medic','support']` |
| `PROFESSIONS` | `Array<Object>` | 职业列表（含显示名，用于 GUI） |

**`PROF_CONFIGS` 结构（每个职业）：**

```js
PROF_CONFIGS.assault = {
  guns: {
    primary: {
      ak47: {
        gunId: 'tacz:ak47',         // TACZ GunId
        GunFireMode: 'AUTO',
        GunCurrentAmmoCount: 30,
        ammo: {                     // ★ 弹药配置（补给模块常用）
          ammoId: 'tacz:762x39',    //   弹药物品 ID
          main: 210,                //   主武器备弹量
          level: 2,                 //   弹药等级（有则发弹药盒）
        },
        attachments: {              // ★ 配件配置
          scope: [{id:'...'}, ...],
          muzzle: [{id:'...'}, ...],
          // ...
        },
      },
    },
    secondary: { /* 同上 */ },
  },
  nonTaczDisplay: { /* 非 TACZ 武器显示映射 */ },
  nonTaczAmmo:   { /* 非 TACZ 弹药映射 */ },
  armor:         [ /* 护甲列表 */ ],
  extras:        [ /* 额外物品 */ ],
}
```

### 构建后的查表数据

| 变量名 | 类型 | 说明 |
|-------|------|------|
| `GUN_TACZ_FLAT` | `Object` | 扁平查表：`weaponId → 完整配置`（含 ammo/attachments） |
| `GUN_TACZ_CONFIG` | `Object` | 按职业/类别分组：`GUN_TACZ_CONFIG.primary.assault.id → config` |
| `PROF_WEAPONS_MAP` | `Object` | 职业武器 ID 列表：`PROF_WEAPONS_MAP.assault = { primary: [...], secondary: [...] }` |
| `VANILLA_WEAPON_DISPLAY` | `Object` | 非 TACZ 武器显示配置 |
| `VANILLA_WEAPON_AMMO` | `Object` | 非 TACZ 弹药配置 |

### 可调用的函数

| 函数签名 | 说明 | 返回值 |
|---------|------|--------|
| `getTaczConfig(weaponId)` | **★ 查武器完整配置**（含 ammo、attachments） | `Object` 或 `null` |
| `getProfessionWeaponList(profession, category)` | 获取职业的武器列表 | `Array` |
| `getProfConfig(id)` | 获取职业基础配置（护甲+额外物品） | `{armor, extras}` 或 `null` |
| `getGunAttachments(player, weaponId)` | 读取玩家已保存的配件配置 | `Object` (slotKey → attachmentId) |
| `setGunAttachment(player, weaponId, slotKey, attachmentId)` | 设置单个配件 | 无返回值 |
| `clearGunAttachments(player, weaponId)` | 清除某把枪所有配件 | 无返回值 |
| `saveGunAttachments(player, weaponId)` | 保存当前配件到 persistentData | 无返回值 |
| `cleanId(raw)` | **★ 清洗 ID**（去引号，常用于 persistentData 读取） | `string` |
| `isTaczGun(wp)` | 判断是否为 TACZ 枪械物品 | `boolean` |
| `resolveTaczGun(cfg)` | 生成 TACZ 枪械物品 | `ItemStack` |
| `giveWeaponAmmo(player, weaponId, slot)` | 发放武器对应弹药 | 无返回值 |
| `giveTaczAmmo(player, weaponId, slot)` | 发放 TACZ 弹药 | 无返回值 |
| `applySavedAttachments(gunStack, player, weaponId)` | 应用已保存的配件到物品 | 无返回值 |
| `getBackpackData(player)` | 读取玩家背包数据 | `Object` |
| `setBackpackData(player, data)` | 写入玩家背包数据 | 无返回值 |

## 弹药补给站

**代码路径：** `server_scripts/sbw_vehicle/ammo_replenish/`

> 注：弹药补给站是**方块驱动**的系统（`kubejs:ammo_crate`），但也暴露了可供其他模块调用的数据。

### 可调用的常量

| 常量名 | 类型 | 说明 |

| `AMMO_KEY_MAP` | `Object` | item ID → 弹药短名（反向映射，自动构建） |

**`AMMO_ID_MAP` 弹药映射表（其他模块补弹药常用）：**

```js
// 短名 → 完整 ID
AMMO_ID_MAP.large_shell_ap  // 'superbwarfare:large_shell_ap'
AMMO_ID_MAP.rifle_ammo      // 'superbwarfare:rifle_ammo'
AMMO_ID_MAP.heavy_ammo      // 'superbwarfare:heavy_ammo'
// ... 详见 a_config.js

// 反向查询
AMMO_KEY_MAP['superbwarfare:large_shell_ap']  // 'large_shell_ap'
```

### 可调用的函数

| 函数签名 | 说明 |
|---------|------|
| `readBlockConfig(block)` | 读取方块 persistentData 中的补给站配置 |
| `getAmmoDisplayName(ammoKey)` | 获取弹药显示名（带颜色代码） |
| `isSBWVehicle(entity)` | 判断是否为 SBW/MCSP 载具 |
| `executeStationReplenish(block, level, ignoreCooldown)` | 执行一次完整的扫描补给流程 |
| `replenishVehicle(entity, slots, level)` | 补给单辆载具的弹药 |

## 典型弹药补给模块调用流程

根据各模块暴露的接口，一个弹药补给模块的标准流程如下：

```js
// 1. 查询武器配置（获取弹药信息）
var cfg = getTaczConfig(weaponId)
// → { gunId, ammo: { ammoId, main, level }, attachments: {...} }

// 2. 查询玩家已装的配件
var attachments = getGunAttachments(player, weaponId)
// → { extended_mag: "tacz:extended_mag_3", scope: "tacz:sight_t1" }

// 3. 如果要查弹药短名→完整ID
var fullAmmoId = AMMO_ID_MAP['rifle_ammo']
// → 'superbwarfare:rifle_ammo'

// 4. 检查模块是否启用
if (isModuleEnabled(server, 'ammo_supply')) {
  // 执行补给逻辑
}

// 5. 使用 cleanId 清洗 persistentData 读出的 ID
var raw = player.persistentData.getString('mainWeapon')
var weaponId = cleanId(raw)
```
