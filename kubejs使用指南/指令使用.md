## 概览

本服务器注册了以下基于 **Brigadier** 的自定义指令：

| 指令 | 用途 | 权限等级 |
|------|------|---------|
| `/module` | 模块开关管理，数据包驱动各模块启用/停用 | OP 2 级 |
| `/profequip` | 发放职业装备（护甲、武器、配件、弹药） | OP 2 级 |
| `/kubejsadmin` | 清空玩家职业/队伍配置 | OP 2 级 |
| `/team_revive` | 管理队伍复活券（增/查/重置） | OP 2 级 |
| `/sbw_vehicle` | 管理 SBW 载具部署（部署/重置/状态查询） | OP 2 级 |

> 这些指令均使用 `ServerEvents.commandRegistry` 注册，**完全兼容**：控制台（服务器终端）、命令方块、数据包（`/function`）、玩家直接输入。

## `/module` — 模块开关管理

### 权限

需要 **OP 2 级** 或以上权限。

### 子命令

```
/module <模块id> on     启用指定模块
/module <模块id> off    停用指定模块
/module list            列出所有模块状态
```

按 Tab 键可自动补全模块 ID。

### 可用模块

| 模块 ID | 说明 |
|---------|------|
| `team_revive` | 队伍复活券系统（死亡扣券、淘汰触发） |
| `team` | 队伍选择器（控制"加入战场"按钮显隐） |

### 数据包调用示例

```mcfunction
# 游戏开始时，启用所需模块
/module team_revive on
/module team on

# 游戏结束时，停用
/module team_revive off
/module team off
```

### 执行来源兼容性

| 来源 | module \<id\> on/off | module list |
|------|:------------------:|:-----------:|
| 玩家（OP 2+） | ✅ | ✅ |
| 控制台 | ✅ | ✅ |
| 命令方块 | ✅ | ✅ |
| 数据包 (`/function`) | ✅ | ✅ |

## `/profequip` — 职业装备发放

### 权限

需要 **OP 2 级** 或以上权限。

### 子命令

```
/profequip give [<targets>]   发放装备
/profequip list               查看所有在线玩家的职业与武器选择
/profequip help               显示帮助信息
```

#### `give` 子命令详情

| 用法 | 说明 |
|------|------|
| `/profequip give` | **仅玩家使用**：给自己发放装备（控制台使用会报错） |
| `/profequip give <targets>` | 给指定目标发放装备，支持选择器 |

当控制台执行时必须指定目标，例如：
```
/profequip give @a
```

#### `list` 子命令

显示所有在线玩家的：
- 玩家名
- 已选职业
- 主武器 / 副武器 / 特殊武器

#### `help` 子命令

显示指令帮助信息。

### 目标选择器

`<targets>` 参数使用 Minecraft 原生的玩家选择器：

| 选择器 | 含义 | 示例 |
|--------|------|------|
| `<玩家名>` | 指定玩家 | `xkmxz2503` |
| `@a` | 所有在线玩家 | `@a` |
| `@p` | 最近的玩家 | `@p` |
| `@s` | 执行者自己 | `@s` |
| `@r` | 随机玩家 | `@r` |

> 注意：本指令使用 `EntityArgument.players()` 参数类型，**仅接受玩家**，如 `@e[type=minecraft:zombie]` 等非玩家选择器会直接报错。

### 执行来源兼容性

| 来源 | give (无参数) | give <targets> | list | help |
|------|:-----------:|:-------------:|:----:|:----:|
| 玩家（OP 2+） | ✅ 给自己 | ✅ | ✅ | ✅ |
| 控制台 | ❌ 报错提示 | ✅ | ✅ | ✅ |
| 命令方块 | ❌ 报错提示 | ✅ | ✅ | ✅ |
| 数据包 (`/function`) | ❌ 报错提示 | ✅ | ✅ | ✅ |

### 发放流程

执行 `/profequip give` 时，按以下顺序处理：

1. **前置检查 ① — 容器守卫**：玩家当前是否有容器打开（GUI、箱子、工作台等）？若有，中断并提示"请先关闭当前打开的界面"
2. **前置检查 ② — 标签守卫**：玩家是否仍有 `no_loadout` 标签？若有，中断并提示"尚未选择完整的职业装备"
3. **前置检查 ③**：玩家是否已选择职业？若未选择，中断并提示
4. **清空背包 + 护甲槽**：保留职业选择器物品
5. **装备护甲**：根据职业配置直接穿到护甲槽
6. **发放主武器**：TACZ 枪械 → 应用已保存的配件 + 弹药盒
7. **发放副武器**：同上
8. **发放特殊武器**：同上（可选）
9. **发放额外物品**：食物、药水等职业配置中的额外物品
10. **清理标签**：成功发放后移除 `no_loadout` 标签

> GUI 内的"给予装备"按钮跳过容器守卫和标签守卫（玩家已在界面内主动操作）。
> 其内部流程为：先关闭 GUI → 等待 5 tick（≈0.25 秒）让客户端库存快照完全恢复 → 再执行装备发放。
> 这种延迟发放避免了 Minecraft GUI 关闭时库存快照覆盖已发放物品的经典问题。
>
> 容器守卫使用 `target.containerMenu !== target.inventoryMenu` 直接检查实时容器状态，
> 不依赖事件回调，100% 准确。

### 使用示例

```mcfunction
# 给自己发放装备（仅玩家）
/profequip give

# 给指定玩家发放
/profequip give xkmxz2503

# 给所有在线玩家发放
/profequip give @a

# 随机选一个玩家发放
/profequip give @r

# 查看所有玩家配置
/profequip list

# 查看帮助
/profequip help

# 从控制台给所有玩家发放
/profequip give @a
```

## `/kubejsadmin` — 管理员管理

### 权限

需要 **OP 2 级** 或以上权限。

### 子命令

```
/kubejsadmin profession <targets>   清空目标玩家的职业选择 + 移除职业标签
/kubejsadmin menu <targets>         清空目标玩家的队伍配置
```

两个子命令**都必须指定目标**，不支持无参数执行。

#### `profession` 子命令

执行以下操作：
1. 移除所有职业标签（`assault`、`scout`、`medic`、`support`）
2. 清除 `persistentData` 中的职业、主武器、副武器、特殊武器数据

#### `menu` 子命令

执行以下操作：
1. 清除 `persistentData` 中的队伍（team）数据

### 执行来源兼容性

| 来源 | profession \<targets> | menu \<targets> |
|------|:-------------------:|:--------------:|
| 玩家（OP 2+） | ✅ | ✅ |
| 控制台 | ✅ | ✅ |
| 命令方块 | ✅ | ✅ |
| 数据包 (`/function`) | ✅ | ✅ |

### 使用示例

```mcfunction
# 清空指定玩家的职业选择（控制台/玩家均可）
/kubejsadmin profession xkmxz2503

# 清空所有在线玩家的职业
/kubejsadmin profession @a

# 清空随机玩家的队伍配置
/kubejsadmin menu @r

# 清空最近的玩家的队伍配置
/kubejsadmin menu @p
```

## `/sbw_vehicle` — 载具部署管理

### 权限

需要 **OP 2 级** 或以上权限。

### 子命令

```
/sbw_vehicle start              激活系统 + 初始化状态 + 启动补员循环
/sbw_vehicle stop               停用系统 + 停止补员循环 + 清除所有载具
/sbw_vehicle deploy [<team>]    部署载具（指定队伍或全部）
/sbw_vehicle redeploy            强制重新部署（清旧+重部署）
/sbw_vehicle reset               重置所有载具状态
/sbw_vehicle clear [<team>]      清除载具实体 + 重置状态（不指定队伍时完全擦除 persistentData）
/sbw_vehicle status              查看各载具当前状态（适配全部8个状态）
/sbw_vehicle timelist            查看所有补员倒计时列表
```

#### `deploy` 子命令

| 用法 | 说明 |
|------|------|
| `/sbw_vehicle deploy` | 部署**所有载具** |
| `/sbw_vehicle deploy <team>` | 部署指定队伍的载具（如 `attack` / `defense`） |

部署时会自动应用配置中的 `deployNBT` 模板（能量、弹药、预装填、部件状态等）。

#### `redeploy` 子命令

执行流程：
1. 先调用 `reset` 清除所有存活的载具实体 + 清空状态记录
2. 重新部署所有载具（含 `deployNBT` 模板）

#### `reset` 子命令

清除所有存活的载具实体，重置所有状态记录至初始。

#### `status` 子命令

显示每辆载具的 8 状态状态机当前状态：`未初始化` / `空闲` / `等待区块` / `区块就绪` / `已部署` / `载具超量` / `载具不足` / `补员中`（含倒计时）。

#### `timelist` 子命令

显示所有正在进行补员倒计时的载具列表及剩余时间。

#### `clear` 调试子命令

| 用法 | 说明 |
|------|------|
| `/sbw_vehicle clear` | 清除**所有队伍**的载具实体并重置状态，**完全擦除 persistentData**（含 active 标志） |
| `/sbw_vehicle clear <team>` | 清除指定队伍的载具实体并重置状态 |

> 清除后需重新 `/sbw_vehicle start` 启动系统。

### 执行来源兼容性

| 来源 | start | stop | deploy | redeploy | reset | clear | status | timelist |
|------|:----:|:----:|:------:|:--------:|:-----:|:-----:|:------:|:--------:|
| 玩家（OP 2+） | ✅ | ✅ | ✅ | ✅ | ✅ | ✅ | ✅ | ✅ |
| 控制台 | ✅ | ✅ | ✅ | ✅ | ✅ | ✅ | ✅ | ✅ |
| 命令方块 | ✅ | ✅ | ✅ | ✅ | ✅ | ✅ | ✅ | ✅ |
| 数据包 (`/function`) | ✅ | ✅ | ✅ | ✅ | ✅ | ✅ | ✅ | ✅ |

### 使用示例

```mcfunction
# 激活系统 + 启动补员循环
/sbw_vehicle start

# 查看所有载具当前状态
/sbw_vehicle status

# 查看补员倒计时列表
/sbw_vehicle timelist

# 部署所有队伍载具
/sbw_vehicle deploy

# 强制重新部署所有载具
/sbw_vehicle redeploy

# 重置所有载具状态
/sbw_vehicle reset

# 停用系统 + 清除所有载具
/sbw_vehicle stop

# 调试：清除进攻方所有载具实体
/sbw_vehicle clear attack
```

> 详细机制说明请参阅 **[SBW 载具自动部署-使用教程.md](./sbw_vehicle-使用教程.md)**。

## 常见问题

### Q: 执行指令提示"没有权限"

A: 需要 OP 2 级或以上。请让服主执行：
```mcfunction
/op <你的玩家名>
```

### Q: 控制台执行 `/profequip give` 提示必须指定目标

A: 控制台没有"自己"的概念，必须加上目标参数：
```mcfunction
/profequip give @a
```

### Q: 提示"未找到有效的目标玩家"

A: Brigadier 的 `players` 参数类型会自动验证目标合法性。请检查：
- 目标玩家是否在线
- 目标名是否拼写正确
- 选择器是否能匹配到玩家

### Q: 提示"请先使用职业选择器选择兵种"

A: 玩家尚未通过 GUI 选择职业。请使用职业选择器物品进行选择，或让管理员检查配置。

### Q: 提示"请先关闭当前打开的界面"

A: 玩家当前有容器/界面打开（职业选择 GUI、箱子、工作台等）。关闭所有打开的界面后重试。

### Q: 提示"尚未选择完整的职业装备"

A: 玩家仍有 `no_loadout` 标签，表示尚未在 GUI 中完成武器配置或尚未领取过装备。请使用职业选择器完成选择。

### Q: 玩家领取装备后仍无法再次领取

A: 如果玩家已关闭 GUI 但仍提示错误，检查是否仍有容器打开（如工作台、箱子）。使用 `/tag <玩家名> list` 查看是否有残留的 `no_loadout` 标签。

### Q: 修改脚本后需要做什么？

A: 执行 `/kubejs reload` 重新加载脚本。如果修改了语言文件（`assets/kubejs/lang/`），还需要按 `F3+T` 刷新资源包。

## `/team_revive` — 队伍复活券管理

### 权限

需要 **OP 2 级** 或以上权限。

### 子命令

``` 
/team_revive add <数量>           为进攻方增加复活券（不超过上限）
/team_revive remove <数量>        为进攻方削减复活券（下限为 0，测试用）
/team_revive reset                重置所有队伍复活券到初始值，清除淘汰标记
/team_revive status [<队伍>]      查看复活券状态（不指定时显示全部）
```

### 使用示例

```mcfunction
# 为进攻方增加 50 张复活券
/team_revive add 50

# 为进攻方削减 30 张复活券（测试用）
/team_revive remove 30

# 查看所有队伍状态
/team_revive status

# 查看指定队伍状态
/team_revive status attacker

# 重置所有复活券
/team_revive reset
```

### 执行来源兼容性

| 来源 | add | reset | status |
|------|:---:|:-----:|:------:|
| 玩家（OP 2+） | ✅ | ✅ | ✅ |
| 控制台 | ✅ | ✅ | ✅ |
| 命令方块 | ✅ | ✅ | ✅ |
| 数据包 (`/function`) | ✅ | ✅ | ✅ |

### 详细说明

- **复活券模型**：每个队伍拥有一个可消耗的复活券池，玩家死亡时消耗 1 张
- **上限保护**：`add` 指令不会使复活券超过配置上限
- **淘汰触发**：券归零时自动调用数据包函数 `game:eliminated`，只触发一次
- **启用模块**：数据包执行 `/module team_revive on` 后才生效

> 详细机制说明请参阅 **[队伍复活券系统-使用教程.md](./team_revive-使用教程.md)**。
