# 模块管理器（Module Manager）使用教程

## 概述

模块管理器通过 `/module` 命令统一控制各 KubeJS 模块的启用/停用。数据包只需执行对应命令即可激活模块功能，替代了原有的计分板检测模式。

## 目录结构

```
server_scripts/module_manager/
├── a_registry.js    ← 模块注册表（新增模块只需在此添加一条记录）
└── main.js          ← 核心逻辑 + /module 命令注册
```

## 命令参考

| 命令 | 说明 | 权限 |
|------|------|------|
| `/module <id> on` | 启用指定模块 | OP 2 |
| `/module <id> off` | 停用指定模块 | OP 2 |
| `/module list` | 列出所有模块状态 | OP 2 |

按 Tab 键可自动补全模块 ID。

## 已注册模块

| 模块 ID | 名称 | 说明 |
|---------|------|------|
| `team_revive` | 队伍复活券 | 控制死亡时是否消耗复活券、触发淘汰 |
| `team` | 队伍选择器 | 控制"加入战场"按钮是否显示 |

## 数据包调用

```mcfunction
# 游戏开始时
/module team_revive on
/module team on

# 游戏结束时
/module team_revive off
/module team off
```

## 新增模块

如需新增受 `/module` 管理的模块，只需在 `a_registry.js` 的 `MODULE_REGISTRY` 中添加一条记录：

```js
{ id: 'your_module_id', name: '显示名称', default: false }
```

然后在对应的脚本中用 `isModuleEnabled(server, 'your_module_id')` 判断模块是否启用。

## 全局 API

| 函数 | 参数 | 说明 |
|------|------|------|
| `isModuleEnabled(server, moduleId)` | server, 模块ID | 判断模块是否已启用 |
| `setModuleEnabled(server, moduleId, enabled)` | server, 模块ID, 布尔值 | 设置模块启用状态 |
