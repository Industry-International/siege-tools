# KubeJS 持久化数据存储说明

> 适用环境：NeoForge 1.21.1 · KubeJS 7

## 一、存储机制概述

本项目使用 `server.persistentData` 进行持久化数据存储。

### 存储位置

```
saves/<世界名>/kubejs/persistent_data.nbt
```

### 与 Minecraft 原版存储的区别

| 特性 | `server.persistentData`（本项目使用） | Minecraft `/data merge storage` |
|------|--------------------------------------|-------------------------------|
| 存储位置 | `saves/<世界>/kubejs/` 目录下 | `level.dat` 内的 `minecraft:persistent` |
| 归属方 | **KubeJS 独立存储** | Minecraft 原版命令存储 |
| 对数据包的影响 | **无影响** | 数据包常用此系统 |
| API 访问方式 | `server.persistentData.putString/getString` | `/data merge storage` 命令 |
| 读写方式 | JSON 字符串读写 | NBT 直接操作 |

**结论：`server.persistentData` 不会污染 Minecraft 原版数据存储系统，不会干扰数据包开发。**

## 二、模块存储结构

### 2.1 SBW 载具系统

**存储键：** `sbw_vehicle`
**代码位置：** `server_scripts/sbw_vehicle/main.js`

```json
{
  "active": true,
  "vehicles": {
    "attack_tank_1": {
      "status": "alive",
      "team": "attack",
      "vehicleType": "superbwarfare:t_90a",
      "uuid": "xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx"
    },
    "defense_tank_1": {
      "status": "respawning",
      "team": "defense",
      "vehicleType": "superbwarfare:t_90a",
      "uuid": "yyyyyyyy-yyyy-yyyy-yyyy-yyyyyyyyyyyy",
      "respawnDelay": 1200,
      "destroyedTick": 1234567
    }
  }
}
```

**字段说明：**

| 字段 | 类型 | 说明 |
|------|------|------|
| `active` | boolean | 系统是否激活 |
| `vehicles` | object | 载具字典，键为 vehicleId |
| `status` | string | `"alive"` / `"respawning"` / `"dead"` |
| `team` | string | `"attack"` / `"defense"` |
| `vehicleType` | string | 实体类型 ID |
| `uuid` | string | 实体 UUID（O(1) 查找用） |
| `respawnDelay` | int | 重生延迟（tick） |
| `destroyedTick` | int | 被摧毁时的服务器 tick |

## 三、读写接口

### 读取

```javascript
function getStore(server) {
  let raw = server.persistentData.getString('存储键')
  if (!raw || raw === '') return {}  // 默认空值
  try { return JSON.parse(raw) } catch(e) { return {} }
}
```

### 写入

```javascript
function saveStore(server, data) {
  server.persistentData.putString('存储键', JSON.stringify(data))
}
```

## 四、设计原则

1. **最小化存储** — 只存储当前运行时状态（如载具存活/重生状态），不存储配置（配置在 `config.js` 中用 `const` 定义）
2. **JSON 序列化** — 所有持久数据以 JSON 字符串形式读写，确保可读性和跨版本兼容性
3. **容错处理** — 读取时检测空值和 JSON 解析异常，自动返回默认空对象
4. **模块隔离** — 每个模块使用独立的存储键，互不干扰

## 五、注意事项

- ❌ **不要**使用 `/data merge storage minecraft:persistent` 来读写本系统的数据
- ❌ **不要**直接修改 `persistent_data.nbt` 文件（会造成数据损坏）
- ✅ 通过 KubeJS 的 `server.persistentData` API 读写
- ✅ 如需调试，可用 `/kubejs hand server.persistentData` 查看（如果装了相关调试工具）
